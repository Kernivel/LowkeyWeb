ABOUT

UE5 Developer & 3D Artist

I build game systems from the ground up — C++ architectures in Unreal Engine 5, AI behaviors, ability pipelines, and networked gameplay — alongside the 3D assets that bring them to life. Hard-surface modelling, PBR texturing, and game-ready production in Blender and Substance Painter.

My work sits at the intersection of tech and art: I implement the systems, then model and texture the assets they depend on — from weapon mechanics to the gun in the player's hands.

Lowkey — my current project — is a cooperative tactical horror FPS with a strong focus on gunplay and atmosphere, set in an SCP-like universe. The game revolves around hunting down artifacts and anomalies as part of a co-op extraction loop — where players drop into degraded environments, recover high-value artefacts, and extract alive. Artefacts fund new equipment and unlock crafted abilities. Losing will set you back a little — just enough to keep you on your toes.

It is where I apply and push everything I know: a GAS-driven NPC alert system, a dual-layer weapon architecture, MetaSound-based audio, and modular hard-surface weapons built in Blender.

UE5 SYSTEMS

Dual-layer architecture separating data from visuals. The UObject layer (WeaponObject stored in InventoryComponent) holds all persistent stats, accessories, and ammo — never spawned into the world. A parallel Actor layer (AWeapon) handles world-space visuals and socket attachment to the character skeleton.

A WeaponComponent on the player acts as a runtime stats mirror — damage, fire rate, fire mode, ammo counts — so GAS, UI, and AI all read from one source without direct coupling to the raw weapon data object.

  • Fire modes — Single Shot / Burst / Automatic
  • Modular accessories — RIS rails, suppressors, scopes, stocks, magazines
  • Recoil via UCurveVector assets (camera + weapon bob, independently tuned)
  • Inventory → DataTable → WeaponObject creation pipeline
  • Save / load via custom PlayerSave serialization
  • MetaSound fire + tail assets with attenuation per weapon
UWeaponObject → UInventoryComponent → AWeapon (Actor)
AWeapon → UWeaponComponent (mirror) → GAS / UI / AI

Fully server-authoritative NPC brain built around a GAS-driven alert state machine. Detection level is a Gameplay Attribute — incremented and decremented by custom Gameplay Effects based on line-of-sight, distance, and elapsed time. No tick-polling.

Alert state transitions replicate to clients via ReplicatedAlertType → OnRep_AlertType, syncing world-space indicator widgets and sounds without authority concerns.

CALM → SUSPICIOUS → PATROLLING → COMBAT
  • AlertComponent — applies GE_AlertIncrease / GE_AlertDecrease, distance-scaled rate, view angle check
  • 50% threshold locks Suspicious state — timed decay prevention prevents jitter
  • 100% threshold → immediate Combat transition via max-alert delegate
  • BTTask_LookAround — 3-phase scan ±60° at 90°/s, investigation on last-known location
  • BTTask_ShootTarget — raycast-based (not projectile), configurable inaccuracy, burst fire support
  • BTTask_ExitSuspicious — resets alert to 25%, returns to Roaming branch
  • Custom BT Decorators — BTDecorator_CheckAlertState (observer aborts), BTDecorator_HasTarget
  • Sub-components: RoamingComponent (random patrol), CombatComponent, AlertComponent

Custom UAbilitySystemComponent extended to expose SendLocalInputPressed / Released to Blueprints, enabling clean input-to-ability binding without native code per ability.

Player ASC lives on ACustomPlayerState for persistence across respawns — initialized in PossessedBy / OnRep_PlayerState. NPC ASC lives directly on the Pawn (ABaseNPC) for lighter overhead.

  • CharacterAttributeSet — Health, MaxHealth (player + NPC)
  • AlertAttributeSet — AlertLevel, MaxAlertLevel (NPC only) — drives full detection pipeline
  • Damage via SetByCaller with tag Data.Damage
  • State tags — Gameplay.State.IsADSing, IsFiring, IsReloading, IsInvisible, NPC.State.Blind
  • FireWeaponAbility — integrates recoil system, ammo consumption, and network authority
  • GA_NPCShoot — NPC-side GAS ability wrapping raycast fire logic
  • All attributes fully replicated with OnRep_ callbacks

State machine driven by Blueprint booleans (ADS, sprint, crouch, reload, fire) feeding into an AnimGraph with per-layer blending. First-person and third-person meshes are fully separate, each with their own Animation Blueprint.

Weapon sway, bob, and recoil are curve-based — authored directly in the Unreal editor as UCurveVector assets and played back procedurally. This allows rapid per-weapon tuning to match the tactile MW2019 reference.

  • Camera recoil + weapon kick via UCurveVector playback, independently controllable
  • ADS blend with FOV transition curve
  • Sprint, crouch, idle, move state blending
  • Character bodies driven by Mutable plugin — runtime mesh customization
  • Hair via UGroomComponent (strand-based groom simulation)

Weapon audio managed through MetaSound graphs for precise layering of fire, tail, and mechanical sounds. MetaSound asset references, attenuation settings, and submix routing are stored directly on UWeaponComponent — keeping audio data co-located with the weapon stats it belongs to.

  • Per-weapon MetaSound assets — fire layer + tail layer independently authored
  • Attenuation and submix routing configurable per weapon in WeaponComponent
  • Multicast RPC ensures fire audio synchronization across all clients

Cooperative architecture — strictly server-authoritative. All gameplay decisions (NPC AI, damage, ability activation) execute on the server; results replicate to clients through properties and RPCs.

Player GAS lives on ACustomPlayerState to survive respawn cycles without ability loss. NPC AI runs server-only on ABaseNPCController; observable results (alert state, animation) replicate as needed.

  • NPC alert state replicated via ReplicatedAlertType + OnRep_AlertType
  • GAS attributes fully replicated with client-side prediction
  • Multicast RPC for networked fire audio
  • Server RPCs for ability activation and weapon actions
  • Team system via IGenericTeamAgentInterface — Player / Monsters / PMC / Neutral
